# 實作計畫：台股未實現損益四捨五入與幣別動態數字格式優化 (v2)

## 目的與問題描述
1. **對齊券商 APP 計算與顯示習慣**：
   目前試算表在計算台股（TWD）未實現損益時，由於包含浮點數（如 `0.1425%` 的手續費與 `0.1%`/`0.3%` 的證交稅），會算出包含小數點的金額（例如 `7292.96`）。然而，台灣券商 APP 上通常僅顯示四捨五入後的整數（例如 `7293`），且台股本身不使用小數。
2. **區分台美股顯示格式**：
   系統目前統一對「未實現損益」、「已實現損益」與「累計總損益」套用雙位數小數點的通用格式 `[Red]$#,##0.00`。
   * *解決方案*：
     * 我們將在 [code.gs](file:///mnt/c/Users/user/Documents/Configuration/stock-pnl/code/code.gs) 中修改台股未實現損益公式，改為**分項 FLOOR ＋外層 ROUND**：預估賣出手續費與證交稅**分別以 `FLOOR()` 無條件捨去到元**（與券商 App 及本專案 Sidebar `calculateFee` 的 `Math.floor` 邏輯完全一致），最後以 `ROUND(..., 0)` 將成本端的浮點尾數收整為整數。
     * 我們將修改數字格式設定邏輯，依幣別分區塊設定：台股的**市值與損益 (F/G/H/I)** 套用 `NT$#,##0` 整數格式，**現價與均價 (C/E) 保留兩位小數**（台股股價本身有小數，如 143.50，整數顯示會誤導）；美股全部套用 `US$#,##0.00` 小數格式。

## 與券商 App 的差異來源分析

| 差異來源 | 常態差異幅度 | 本計畫是否解決 |
|---|---|---|
| 手續費/證交稅捨去方式（App 分項無條件捨去 vs 原公式合併浮點計算） | 每檔約 1~2 元 | ✅ 分項 `FLOOR` 後與 App 算法同構 |
| 未實現損益的小數尾數（顯示） | < 1 元 | ✅ `ROUND` 後為整數，顯示與 App 一致 |
| `GOOGLEFINANCE` 現價延遲（最多約 20 分鐘）vs App 即時價 | 隨股價波動，可達數十~數百元 | ❌ 結構性限制，無法以捨入解決 |
| 移動平均成本的浮點尾數（賣過一部分的持股） | < 1 元 | ✅ 由外層 `ROUND` 收整 |

## 預計修改內容

### [MODIFY] [code.gs](file:///mnt/c/Users/user/Documents/Configuration/stock-pnl/code/code.gs)

**1. 修改未實現損益公式（實際位置：code.gs 第 758-759 行）：**
台股改為「分項 FLOOR ＋外層 ROUND」：手續費與證交稅各自 `FLOOR()` 捨去到元（與券商 App 及 Sidebar `calculateFee` 一致），最外層 `ROUND(..., 0)` 收整成本端浮點尾數。美股維持原式（`sellCostRate` 為 0，即 `F - D*E`）。**注意：此迴圈中原本沒有 `isTw` 變數，必須在迴圈內自行定義**（從 `holdings[i].info.currency` 判斷），否則會 ReferenceError：

```diff
       // 扣除預估賣出手續費與證交稅以對齊券商 APP 淨損益,僅台股適用
       // (一般股票 0.3%,ETF 代號 00 開頭 0.1%;美股多為零手續費,不預扣)
-      const sellCostRate = 'IF(LEFT($A' + r + ',4)="TPE:",' + feeRate + '+IF(MID($A' + r + ',5,2)="00",0.001,0.003),0)';
-      fUnreal.push(['=IF(ISNUMBER($C' + r + '),F' + r + '-D' + r + '*E' + r + '-F' + r + '*' + sellCostRate + ',"")']);
+      // 台股:手續費與證交稅分項 FLOOR 捨去到元(與券商 App / Sidebar calculateFee 同構),
+      // 外層 ROUND 收整移動平均成本的浮點尾數;美股:多為零手續費,不預扣
+      const isTw = holdings[i].info.currency === "TWD";
+      if (isTw) {
+        const taxRate = 'IF(MID($A' + r + ',5,2)="00",0.001,0.003)'; // ETF(00 開頭) 0.1%,一般股票 0.3%
+        fUnreal.push(['=IF(ISNUMBER($C' + r + '),ROUND(F' + r + '-D' + r + '*E' + r +
+          '-FLOOR(F' + r + '*' + feeRate + ')-FLOOR(F' + r + '*' + taxRate + '),0),"")']);
+      } else {
+        fUnreal.push(['=IF(ISNUMBER($C' + r + '),F' + r + '-D' + r + '*E' + r + ',"")']);
+      }
```

> [!NOTE]
> 台股列的代號必為 `TPE:` 前綴（`currency === "TWD"` 即由此判定），故公式內的 `MID($A,5,2)` ETF 判斷安全；原本的 `LEFT($A,4)="TPE:"` 外層防護由 `isTw` 分支取代，不再需要。

**2. 修改數字格式設定邏輯（實際位置：code.gs 第 774-783 行「---- 數字格式 ----」區塊）：**
移除原先整欄套用小數點的 C/E/F/G-I 格式，改為**依幣別分兩個連續區塊**設定（`holdings` 已按 TWD 在前、USD 在後排序）。直接以下列內容取代原區塊，不需動到公式迴圈：

```javascript
  // ---- 數字格式 ----
  const endRow = Math.max(DATA_START + holdings.length - 1, 120);
  dbSheet.getRange("C7:J" + endRow).setHorizontalAlignment("right");
  dbSheet.getRange("D7:D" + endRow).setNumberFormat("#,##0");
  dbSheet.getRange("J7:J" + endRow).setNumberFormat('[Red]0.00%;[Green]-0.00%;0.00%');
  dbSheet.getRange("K7:K" + endRow).setHorizontalAlignment("center").setFontColor("#64748b");

  // 依幣別分區塊設定金額格式(holdings 已按 TWD 在前、USD 在後排序,各為連續區塊)
  // 台股:現價/均價保留兩位小數(股價本身有小數),市值與損益取整數;美股:全部兩位小數
  const twCount = holdings.filter(function (h) { return h.info.currency === "TWD"; }).length;
  const usCount = holdings.length - twCount;
  if (twCount > 0) {
    dbSheet.getRange(DATA_START, 3, twCount, 1).setNumberFormat('"NT$"#,##0.00'); // C 現價
    dbSheet.getRange(DATA_START, 5, twCount, 1).setNumberFormat('"NT$"#,##0.00'); // E 均價
    dbSheet.getRange(DATA_START, 6, twCount, 1).setNumberFormat('"NT$"#,##0');    // F 市值
    dbSheet.getRange(DATA_START, 7, twCount, 3)                                   // G:I 損益
      .setNumberFormat('[Red]"NT$"#,##0;[Green]-"NT$"#,##0;"NT$"0');
  }
  if (usCount > 0) {
    const usStart = DATA_START + twCount;
    dbSheet.getRange(usStart, 3, usCount, 1).setNumberFormat('"US$"#,##0.00');    // C 現價
    dbSheet.getRange(usStart, 5, usCount, 2).setNumberFormat('"US$"#,##0.00');    // E:F 均價/市值
    dbSheet.getRange(usStart, 7, usCount, 3)                                      // G:I 損益
      .setNumberFormat('[Red]"US$"#,##0.00;[Green]-"US$"#,##0.00;"US$"0.00');
  }
```

> [!NOTE]
> **顯示與實值說明**：H 欄（已實現損益）為含小數的靜態值、I 欄 = `SUM(G,H)`，對其套整數格式僅是「顯示」上的四捨五入，儲存格實際值不變；G 欄公式已 `ROUND` 為整數，故摘要卡 C2（`SUMIFS` 加總 G 欄）必為整數，無需另行處理。
